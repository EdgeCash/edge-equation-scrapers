# data/

**Sandboxed outputs.** Anything generated for offline auditing — explicitly NOT served by Vercel and NOT consumed by `web/`.

The contrast with [`public/data/`](../public/README.md) is deliberate:

- `public/data/` → live, user-facing. Serves the website. Anything here can be seen.
- `data/` → sandbox. Audit material. Nothing here is publication-ready.

## Subdirectories

| Path | Purpose |
|------|---------|
| `experimental/mlb-props/` | Daily player-prop projections (Pitcher Ks, Batter Hits, Batter Total Bases). Generated by `run_mlb_props_experimental.py`. EXPERIMENTAL — see [`docs/BRAND_GUIDE.md`](../docs/BRAND_GUIDE.md) Sandbox section. |
| [`backfill/mlb/`](backfill/README.md) | Multi-season historical games + per-game boxscores. One-time bulk pull via `run_mlb_backfill.py`. Used for calibration refits, multi-season Brier validation, and prop backtest grading — all offline. |

## Promotion path

Sandboxed work moves out of `data/experimental/` into `public/data/` **only** when its market clears the same gate as live markets:

- ≥+1% ROI over a rolling 200+ bet backtest
- Brier <0.246 over the same window

Until both are met, the sandbox stays sandboxed. Removing a sandboxed market that doesn't earn its way out is a normal, expected outcome — building the pipeline doesn't entitle anything to publication.
